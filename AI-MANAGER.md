## 任务检查规范

> **重要**：检查任务完成时，不能只看文件是否存在，必须检查代码实现是否完整

### 检查清单

1. **文件存在性检查**
   - Entity 是否存在
   - Mapper 是否存在
   - Service/ServiceImpl 是否存在
   - Controller 是否存在
   - DTO 是否存在

2. **代码完整性检查**
   - Service 是否包含完整 CRUD 方法：
     - `pageQuery()` 分页查询
     - `getDetail()` 获取详情
     - `create()` 创建
     - `update()` 更新
     - `delete()` 删除
   - Controller 是否包含对应 API 端点
   - 业务逻辑是否正确实现（如联动、状态流转等）

3. **实现逻辑审查（关键！）**
   - **必须阅读代码内容，检查逻辑是否正确**
   - 检查方法是否有实际实现，不是空方法或只返回null
   - 检查业务逻辑是否符合设计文档要求
   - 检查是否有明显的bug或遗漏
   - 检查数据库操作是否正确（增删改查）
   - 检查事务注解是否正确使用
   - 检查异常处理是否完善

4. **编译验证**
   - 运行 `mvn compile` 确保无编译错误

### 审查示例

```bash
# 错误示例：只检查文件是否存在
ls backend/src/main/java/com/duoduo/jxc/service/impl/XxxServiceImpl.java
# 文件存在，但不代表代码正确！

# 正确示例：必须阅读代码内容
cat backend/src/main/java/com/duoduo/jxc/service/impl/XxxServiceImpl.java
# 检查：
# 1. create() 方法是否真的保存了数据？
# 2. update() 方法是否真的更新了数据？
# 3. 业务逻辑是否完整？是否有遗漏？
```

### 警告

> **严重警告**：只检查文件名/方法名而不检查实现逻辑，将被开除！
> 管理者必须对代码质量负责，不能敷衍了事。
