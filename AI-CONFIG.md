## 管理者角色（Manager-AI）

> **重要**：项目设置 1 个管理者AI，不参与开发，专职监督和派工

| 角色 | 职责 | 权限 |
|------|------|------|
| **Manager-AI** | 监督进度、派工、协调、汇报 | ✅ 查看所有文件<br>✅ 分配任务<br>✅ 协调跨模块问题<br>❌ 不能修改代码<br>❌ 不负责开发 |

**管理者身份文件**：`AI-MANAGER.md`

**任命方式**：
```
你被任命为项目的 Manager-AI（管理者AI）。
请创建 AI-MANAGER.md 文件，开始工作。
```

## 如何使用

### 步骤1：领取任务

在你的会话开始时，执行以下命令：

```
请帮我领取 AI-X 模块（模块名称），我是 [你的AI名称]
```

### 步骤2：创建身份文件

将此文件复制为 `AI-IDENTITY-{AI_ID}.md` 并填写：

```yaml
# trae 创建 AI-IDENTITY-trae.md
AI_ID: "trae"
AI_NAME: "Trae"
MODULE: "基础数据、销售、工序、质量"
START_TIME: "2026-03-29 17:30"

# codebuddy 创建 AI-IDENTITY-codebuddy.md  
AI_ID: "codebuddy"
AI_NAME: "CodeBuddy"
MODULE: "采购、库存、生产、财务"
START_TIME: "2026-03-29 17:30"
```

### 步骤3：阅读必要文档

**启动时必须按顺序阅读：**

```
1. docs/多AI协作开发规范.md      # 协作规则（必读）
2. docs/split/00-索引与总览.md    # 项目整体结构（必读）
3. docs/split/02-数据模型设计.md  # 数据模型（必读）
4. docs/split/{你的模块文档}.md   # 负责模块的设计文档
```

**各AI对应模块文档：**

| AI_ID | 主要文档 |
|-------|---------|
| trae | `docs/split/12-销售管理模块.md` + `docs/split/02-数据模型设计.md`(基础数据部分) |
| codebuddy | `docs/split/13-采购管理模块.md` + `docs/split/14-库存管理详细设计.md` |

### 步骤4：遵循规范

所有AI必须遵循 `docs/多AI协作开发规范.md`

## 接口依赖说明

### 你提供的接口（其他AI需要调用）

请在你开发的模块完成后，在此记录对外提供的接口：

```yaml
# 示例：AI-2 销售模块提供的接口
provided_apis:
  - POST /api/v1/sales/orders         # 创建销售订单
  - GET  /api/v1/sales/orders/{id}    # 获取订单详情
  - PUT  /api/v1/sales/orders/{id}/status  # 更新订单状态
```

### 你依赖的接口（需要其他AI提供）

请记录你需要调用的其他模块接口：

```yaml
# 示例：AI-2 销售模块依赖的接口
required_apis:
  - from: AI-1
    apis:
      - GET /api/v1/products/{id}     # 获取商品信息
      - GET /api/v1/customers/{id}    # 获取客户信息
  - from: AI-4
    apis:
      - POST /api/v1/inventory/reserve  # 库存占用
  - from: AI-8
    apis:
      - POST /api/v1/receivables       # 创建应收账款
```
