## 五、浏览器操作指南

### 5.1 快速测试路径

#### 路径 1: 查看预订单列表
```
1. 打开 http://localhost:3000
2. 登录（开发环境默认 admin/admin123）
3. 左侧菜单 → 销售 → 销售预定
4. 查看"按单据"列表
```

#### 路径 2: 新增预订单
```
1. 在预订单列表页
2. 点击"新增预定单"按钮
3. 选择客户（下拉框）
4. 点击"选择商品" → 添加商品
5. 输入数量、单价
6. 点击"保存 (Ctrl+Q)"
```

#### 路径 3: 审核预订单
```
1. 在列表中勾选要审核的单据
2. 点击"审核"按钮
3. 看到成功提示
4. 状态变为"已审核"
```

#### 路径 4: 转销售订单
```
1. 找到已审核的预订单
2. 点击"转销售"按钮
3. 系统自动生成销售订单
4. 跳转到销售订单详情页
```

## 七、测试结论

### 7.1 总体评价
销售预定模块核心功能完备，用户体验良好，可以满足日常业务需求。主要问题集中在辅助功能（导出、导入、扫码）尚未实现，但不影响主流程使用。

### 7.2 风险提示
- ⚠️ **并发风险**: 多个销售员同时操作可能导致库存超卖
- ⚠️ **数据一致性**: 缺少乐观锁机制，可能覆盖他人修改

### 7.3 上线建议
- ✅ **可以上线**: 核心功能稳定，风险可控
- ⚠️ **需告知用户**: 导出等功能将在后续版本提供
- 🔧 **尽快修复**: 建议在 1 周内完成并发控制优化
