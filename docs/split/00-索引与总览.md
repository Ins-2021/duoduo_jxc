## 一、项目概述

### 1.1 系统背景

| 系统 | 技术栈 | 定位 | 完成度 |
|------|--------|------|--------|
| duoduo_jxc | Spring Boot + Vue 3 | 进销存/ERP | 85% |
| coding (衣多多) | Django + Vue 3 + 小程序 | 生产管理/MES | 90% |

### 1.2 整合目标

将两系统整合为**完整的服装行业全链路ERP系统**，覆盖：
- 销售管理（批发/零售）
- 采购管理
- 库存管理
- 财务管理
- 生产管理
- 质量管理
- 成本核算

### 1.3 技术选型

| 层级 | 技术 |
|------|------|
| 后端 | Spring Boot 3.2 + MyBatis-Plus + Flowable |
| 前端 | Vue 3 + Element Plus + TypeScript |
| 数据库 | MySQL 8.0 |
| 缓存 | Redis |
| 小程序 | uni-app |

### 7.2 工序分类

1. **裁剪准备**: 验布→铺布→划样→裁剪→验片→分包
2. **缝制前道**: 粘衬→打省→收褶→拼合
3. **缝制主道**: 前幅→后幅→袖子→领子→组装
4. **缝制后道**: 锁眼→钉扣→套结→装饰线
5. **整理工序**: 剪线头→整烫→质检→包装
6. **特殊工序**: 刺绣→印花→洗水→成衣染色

## 八、版本修订记录

### v2.3 (2026-03-28) 修正版

根据 **Trae** 和 **CodeBuddy** 的分析报告，修正以下设计缺陷：

| 修正项 | 类型 | 说明 |
|--------|------|------|
| BOM矩阵设计 | 新增 | 添加`jxc_bom_color_mapping`和`jxc_bom_size_mapping`表，解决"同款不同色码用料不同"问题 |
| Flowable→状态机 | 重构 | 扎包工序流转改用轻量级状态机，避免高并发性能问题 |
| WIP库存逻辑 | 修正 | 在制品不写入统一库存表，改用动态视图统计 |
| 金额精度 | 修正 | 单价字段从`DECIMAL(10,2)`改为`DECIMAL(10,4)` |
| 字段命名 | 统一 | `created_at`→`create_time`，`updated_at`→`update_time` |
| 成本核算详细设计 | 新增 | 添加完整的成本归集、分摊、废品处理设计（第26章） |
| 工资管理模块 | 新增 | 添加计件汇总、工资单生成发放流程（第27章） |
| 消息实时推送 | 新增 | 添加小程序订阅消息推送设计（第43章） |
| 重复内容删除 | 修正 | 删除第6章重复的接口定义 |
