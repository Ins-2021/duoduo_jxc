## 发现的问题明细

### 问题1: 收支记录查询SQL错误 ✅ 已修复

**问题描述**: 
收支记录查询接口返回SQL错误：`Unknown column 'created_by' in 'field list'`

**修复方案**:
在`IncomeExpense`实体类中添加了`@TableField`注解：
```java
@TableField("create_by")
private Long createBy;
@TableField("update_by")
private Long updateBy;
```

**验证结果**: 
重新编译部署后，收支记录查询接口正常工作，返回2条收支记录。

### 问题2: 账户余额未随收支变动更新 (严重)

**问题描述**: 
创建收入单和支出单后，账户余额未发生变化。

**测试步骤**:
1. 账户1原有余额: 10,000.00
2. 新增收入单500元
3. 新增支出单200元
4. 查询账户1余额: 仍为10,000.00

**预期结果**: 
账户余额应变为: 10,000 + 500 - 200 = 10,300

**实际结果**:
账户余额仍为: 10,000.00

**根本原因**: 
`IncomeExpenseService`实现中，收支单的创建和审批流程没有更新关联账户的余额字段。

**修复建议**:
在`IncomeExpenseServiceImpl`中，当收入单审核通过或支出单审核通过时，应同时更新`FinanceAccount`的`balance`字段。

### 问题3: 核销记录未生成 (中等)

**问题描述**: 
执行应收/应付核销操作后，核销记录表`jxc_write_off`中无数据。

**测试步骤**:
1. 执行应收核销操作（部分2000 + 全额3000）
2. 查询核销记录列表，返回空

**预期结果**: 
应在`jxc_write_off`表中生成2条核销记录

**实际结果**: 
表中无数据

**根本原因**: 
`ReceivableServiceImpl.writeOff()`方法只更新了应收账款的已收金额和状态，但没有在`jxc_write_off`表中插入核销记录。
