## 三、数据库设计

### 3.1 新增字段

```sql
-- jxc_cut_bundle 表新增字段
ALTER TABLE jxc_cut_bundle ADD COLUMN qr_code VARCHAR(500) COMMENT '二维码图片URL';
ALTER TABLE jxc_cut_bundle ADD COLUMN qr_data TEXT COMMENT '二维码内容JSON';
ALTER TABLE jxc_cut_bundle ADD COLUMN printed TINYINT DEFAULT 0 COMMENT '是否已打印';
ALTER TABLE jxc_cut_bundle ADD COLUMN printed_at DATETIME COMMENT '打印时间';

-- jxc_bundle 表新增字段  
ALTER TABLE jxc_bundle ADD COLUMN current_location VARCHAR(100) COMMENT '当前位置';
ALTER TABLE jxc_bundle ADD COLUMN scan_count INT DEFAULT 0 COMMENT '扫码次数';
ALTER TABLE jxc_bundle ADD COLUMN last_scan_at DATETIME COMMENT '上次扫码时间';
ALTER TABLE jxc_bundle ADD COLUMN last_scan_by BIGINT COMMENT '上次扫码人';

-- 新增扫码记录表
CREATE TABLE jxc_scan_record (
    record_id BIGINT PRIMARY KEY AUTO_INCREMENT,
    bundle_id BIGINT NOT NULL COMMENT '扎包ID',
    bundle_no VARCHAR(50) NOT NULL COMMENT '扎包号',
    process_id BIGINT COMMENT '工序ID',
    worker_id BIGINT NOT NULL COMMENT '工人ID',
    quantity INT NOT NULL COMMENT '数量',
    scan_type ENUM('QR', 'MANUAL', 'BARCODE') DEFAULT 'QR' COMMENT '扫码类型',
    device_info VARCHAR(200) COMMENT '设备信息',
    ip_address VARCHAR(50) COMMENT 'IP地址',
    scan_at DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '扫码时间',
    UNIQUE KEY uk_bundle_worker_process (bundle_id, worker_id, process_id, DATE(scan_at))
);
```

## 四、API接口设计

### 4.1 二维码相关接口

```
POST /production/cut-bundle/{id}/generate-qr
响应: { qrCodeUrl: "...", qrData: {...} }

POST /production/cut-bundle/{id}/print
响应: { success: true }

POST /bundles/{id}/generate-qr  
响应: { qrCodeUrl: "...", qrData: {...} }

POST /bundles/scan
请求: { qrContent: "...", processId: 1, workerId: 1, quantity: 20 }
响应: { success: true, recordId: 1 }

GET /bundles/{id}/flow
响应: { bundle: {...}, processes: [...] }
```

## 五、安全与性能设计

### 5.1 防重控制
- Redis分布式锁，Key: `scan:lock:{bundleId}:{workerId}`
- 数据库唯一索引防重复
- 扫码间隔限制（同一工人同一扎包30秒内不能重复扫码）

### 5.2 并发控制
- 扎包状态变更使用乐观锁（version字段）
- 计件数量使用原子操作
- 队列异步处理统计更新

### 5.3 二维码安全
- 二维码内容签名验证
- 有效期控制（24小时）
- 防止二维码被篡改
