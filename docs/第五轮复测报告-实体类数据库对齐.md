## 三、仍存在不对齐的项

### 3.1 ❌ SizeCategory (jxc_size_category) - ID 列名不匹配

**问题**: V2026.03.31 的 `CREATE TABLE IF NOT EXISTS jxc_size_category` 使用 `size_category_id` 作为主键，而实体类使用 `@TableId` 标注在 `categoryId` 上，映射为 `category_id`。

- 实体类主键: `categoryId` → 映射列 `category_id`
- DDL 主键: `size_category_id`

**但如果表已存在**（V1.1 创建或更早版本创建），则主键列名取决于原始表结构。迁移脚本使用 `IF NOT EXISTS`，如果表已存在则不会执行 CREATE TABLE。需要检查实际数据库中的列名。

**修复建议**: 确认实际数据库表的主键列名。如果为 `size_category_id`，需要在实体类中添加 `@TableId(value = "size_category_id")`。

### 3.2 ❌ SizeCategory (jxc_size_category) - name vs category_name

- 实体类字段: `private String name` → 映射列 `name`
- DDL 列名: `category_name`

**修复建议**: 在实体类 `name` 字段添加 `@TableField("category_name")`，或修改 DDL 列名为 `name`。

### 3.15 ⚠️ FirstArticleConfirmation (jxc_first_article_confirmation) - deleted 字段冲突

实体类继承了 `BaseEntity`，但 `@TableField(exist = false)` 覆盖了 `deleted` 字段，说明数据库表中确实没有 `deleted` 列。DDL 中也没有 `deleted` 列。**这是正确的处理方式**。

### 3.16 ⚠️ DefectRecord (jxc_defect_record) - deleted 字段冲突

同上，`@TableField(exist = false)` 正确覆盖了 BaseEntity 的 `deleted` 字段。DDL 中有 `deleted` 列，但实体类标记为 `exist = false`。

**问题**: DDL 有 `deleted TINYINT DEFAULT 0` 列，但实体类标记 `@TableField(exist = false)` 意味着不会操作该列。这会导致：
- 逻辑删除不生效（查询不会过滤已删除记录）
- 无法通过 MP 的 deleteById 逻辑删除

**修复建议**: 移除 `@TableField(exist = false)` 标记，让 BaseEntity 的 `@TableLogic deleted` 正常工作。
