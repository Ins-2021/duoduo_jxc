# 数据库变更管理规范

## 一、变更流程

### 1.1 新增表

```sql
-- 步骤1：创建DDL脚本
-- 文件命名：V{版本号}__create_{表名}.sql
-- 示例：V1.0.1__create_jxc_sales_order.sql

CREATE TABLE jxc_sales_order (
    order_id BIGINT PRIMARY KEY AUTO_INCREMENT,
    order_no VARCHAR(50) NOT NULL,
    -- 必备审计字段
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    create_by BIGINT,
    update_by BIGINT,
    deleted TINYINT DEFAULT 0,
    
    UNIQUE KEY uk_order_no (order_no),
    INDEX idx_customer_id (customer_id)
) COMMENT '销售订单表';

-- 步骤2：更新 AI-DATABASE-CHANGELOG.md
```

### 1.2 修改表

```sql
-- 文件命名：V{版本号}__alter_{表名}_{变更内容}.sql
-- 示例：V1.0.2__alter_jxc_sales_order_add_remark.sql

ALTER TABLE jxc_sales_order 
ADD COLUMN remark VARCHAR(500) COMMENT '备注' AFTER status;

-- 步骤2：通知所有AI变更内容
```

### 1.3 删除字段

```sql
-- 删除前必须确认无引用
-- 步骤1：在 AI-DATABASE-CHANGELOG.md 中申请
-- 步骤2：Arch-AI 审核确认
-- 步骤3：执行删除

ALTER TABLE jxc_sales_order DROP COLUMN old_field;
```

## 四、命名规范

### 4.1 表命名

```
jxc_{模块}_{业务}

示例：
jxc_sales_order      - 销售订单
jxc_purchase_order   - 采购订单
jxc_inventory        - 库存
jxc_production_order - 生产订单
```

### 4.2 字段命名

```
主键：{表名单数}_id，如 order_id
外键：{关联表名}_id，如 customer_id
状态：status
时间：xxx_time，如 create_time, delivery_time
金额：xxx_amount，如 total_amount
数量：xxx_qty，如 quantity, available_qty
```

### 4.3 索引命名

```
唯一索引：uk_{字段名}
普通索引：idx_{字段名}
组合索引：idx_{字段1}_{字段2}
```

## 五、审核流程

```
AI提交变更申请
      │
      ▼
Arch-AI 审核DDL ──→ 不通过 → 退回修改
      │
      ▼ 通过
PM-AI 确认影响 ──→ 有冲突 → 协调处理
      │
      ▼ 无冲突
执行DDL脚本
      │
      ▼
更新变更日志
      │
      ▼
通知所有AI
```
