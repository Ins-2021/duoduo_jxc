## 七、修复建议

### 7.1 数据库迁移脚本

```sql
-- 1. 为 jxc_process_record 表添加 record_no 字段
ALTER TABLE jxc_process_record ADD COLUMN record_no VARCHAR(32) DEFAULT NULL COMMENT '记录编号';

-- 2. 为 jxc_first_article_confirmation 表添加 deleted 字段
ALTER TABLE jxc_first_article_confirmation ADD COLUMN deleted TINYINT DEFAULT 0 COMMENT '删除标记';

-- 3. 为 jxc_defect_record 表添加 deleted 字段
ALTER TABLE jxc_defect_record ADD COLUMN deleted TINYINT DEFAULT 0 COMMENT '删除标记';
```

### 7.2 需要创建的表

1. `capacity_alert` - 产能预警表
2. `jxc_process_exception` - 工序异常表

### 7.3 微信登录问题

检查 `AuthController.loginByWechat` 方法中默认账号 "worker" 是否在数据库中存在及密码是否正确。

## 八、测试结论

系统管理模块功能基本正常，CRUD操作均通过测试。小程序端接口存在6个问题需要修复，主要为数据库表结构不完整导致。建议优先修复高严重程度的数据库问题后重新测试。
