### 2.2 year_month SQL保留字 ❌ 仍有问题

```bash
curl -X POST "http://localhost:8080/api/v1/wage/sheet" \
  -H "Authorization: Bearer {token}" \
  -d '{"yearMonth":"2026-03",...}'
```

**错误信息**:
```
You have an error in your SQL syntax; check the manual that corresponds to your MySQL server version for the right syntax to use near 'year_month, start_date, end_date...'
```

**分析**:
- 数据库列名 `year_month` 是MySQL 8.0保留字
- 实体类使用了 `@TableField("year_month")` 但MyBatis-Plus未自动转义
- 需要在字段注解上添加转义处理或在全局配置中设置

**解决方案**: 在 `PayrollSheet.java` 的 `yearMonth` 字段上添加转义:
```java
@TableField("`year_month`")
private String yearMonth;
```

### 2.3 菜单权限 ❌ 仍有问题

| 接口 | 测试结果 | 说明 |
|------|----------|------|
| `/bom/list` (POST) | ❌ 403无权限 | 权限标识不匹配 |
| `/production/order/page` (GET) | ❌ 403无权限 | 权限标识不匹配 |
| `/sales/page` (GET) | ✅ 200成功 | 销售订单正常 |

**权限标识不匹配问题分析**:

| 位置 | 使用的权限标识 |
|------|----------------|
| Java代码 `@PreAuthorize` | `bom:view`, `production:order:view` |
| 数据库 `jxc_sys_menu.perms` | `data:bom:view`, `data:production:order:view` |

**验证**:
```sql
-- 数据库中的权限标识
SELECT menu_id, perms FROM jxc_sys_menu WHERE perms LIKE '%bom%';
-- 结果: data:bom:view, data:bom:add, data:bom:edit, data:bom:delete

-- 但Java代码检查的是
@PreAuthorize("@perm.has('bom:view')")  -- 查找 bom:view
```

**解决方案** (二选一):
1. 修改Java代码: `@PreAuthorize("@perm.has('data:bom:view')")`
2. 修改数据库: 将 `data:bom:view` 改为 `bom:view`
