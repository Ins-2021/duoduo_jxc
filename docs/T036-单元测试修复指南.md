```yaml
  configuration:

  database-schema-update: true
  database-type: mysql

  history-level: full
```

#### 步骤2：准备测试数据库
1. 创建测试数据库（如果不存在）：
```sql
CREATE DATABASE IF NOT EXISTS duoduo_jxc_test 
CHARACTER SET utf8mb4

```

2. 测试数据库应使用与生产相同的表结构

#### 步骤3：验证测试
```bash
cd /Users/zxh/mycoding/duoduo_jxc/backend
mvn clean test -Dtest=StockInOutServiceTest
```

### 注意事项
- ❌ 不要使用H2数据库
- ❌ 不要修改flowable.database-type为h2
- ✅ 使用MySQL数据库连接

## 原问题描述
81个单元测试全部失败，ApplicationContext无法加载

## 错误分析
从测试报告可以看出：
- ApplicationContext failure threshold (1) exceeded
- 所有测试类都无法加载Spring上下文
- 可能原因：MySQL连接配置、Bean创建失败、依赖缺失

## 修复方案（使用MySQL）

### 步骤1：检查application-test.yml配置

文件位置：`backend/src/test/resources/application-test.yml`

**当前配置**：
```yaml
spring:
  datasource:
    driver-class-name: org.h2.Driver  # ❌ 错误，应该使用MySQL
    url: jdbc:h2:mem:testdb
```

**正确配置**：
```yaml
spring:
  datasource:
    driver-class-name: com.mysql.cj.jdbc.Driver
    url: jdbc:mysql://127.0.0.1:3306/duoduo_jxc_test?useUnicode=true&characterEncoding=utf8&useSSL=false&serverTimezone=Asia/Shanghai
    username: root
    password: your_password  # 根据实际情况修改
    hikari:
      maximum-pool-size: 5
      minimum-idle: 1
      connection-timeout: 20000
      
  sql:
    init:
      mode: never  # 测试环境不自动执行schema.sql/data.sql
      
mybatis-plus:
  configuration:
    log-impl: org.apache.ibatis.logging.nologging.NoLoggingImpl

flowable:
  database-schema-update: true
  database-type: mysql  # ✅ 保持mysql
  async-executor-activate: false
  history-level: full
```

### 步骤2：准备测试数据库

1. 创建测试数据库：
```sql
CREATE DATABASE IF NOT EXISTS duoduo_jxc_test 
CHARACTER SET utf8mb4 
COLLATE utf8mb4_unicode_ci;
```

2. 确保测试数据库表结构与生产一致：
```bash
# 从生产数据库导出结构
mysqldump -u root -p --no-data duoduo_jxc > schema.sql

# 导入测试数据库
mysql -u root -p duoduo_jxc_test < schema.sql
```

### 步骤3：检查pom.xml依赖

确保MySQL驱动依赖存在：
```xml
<dependency>
    <groupId>mysql</groupId>
    <artifactId>mysql-connector-java</artifactId>
    <version>8.0.33</version>
</dependency>
```

### 步骤4：检查测试类配置

测试类应该使用：
```java
@SpringBootTest
@ActiveProfiles("test")
@TestPropertySource(properties = {
    "spring.datasource.url=jdbc:mysql://127.0.0.1:3306/duoduo_jxc_test"
})
public class SomeServiceTest {
    // ...
}
```

### 步骤5：清理并重新测试

```bash
cd /Users/zxh/mycoding/duoduo_jxc/backend
mvn clean test -Dtest=StockInOutServiceTest
```

先测试单个测试类，确认基础配置正确后再运行全部测试。

## 备选方案：使用内存数据库但不修改业务代码

如果必须使用MySQL但测试环境无法连接，可以：

### 方案A：使用TestContainers
```xml
<dependency>
    <groupId>org.testcontainers</groupId>
    <artifactId>mysql</artifactId>
    <version>1.19.3</version>
    <scope>test</scope>
</dependency>
```

### 方案B：跳过测试
```bash
mvn clean install -DskipTests
```

**注意**：跳过测试只是临时方案，不代表问题已修复。

## 调试技巧

1. **查看详细错误日志**：
   ```bash
   cat backend/target/surefire-reports/*.txt | grep -A 50 "ERROR"
   ```

2. **检查Bean创建失败**：
   ```bash
   mvn test 2>&1 | grep -i "bean\|create\|error" | head -20
   ```

3. **检查数据库连接**：
   ```bash
   # 手动测试连接
   mysql -h 127.0.0.1 -P 3306 -u root -p -e "SELECT 1"
   ```

4. **使用DEBUG模式**：
   ```bash
   mvn test -X 2>&1 | grep -i "datasource\|connection" | head -20
   ```
