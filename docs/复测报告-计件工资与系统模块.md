## 三、新发现的问题

### 问题1: ProductionDashboardServiceImpl 使用不存在的字段

**问题描述**: `ProductionDashboardServiceImpl.java` 第214行使用了 `record_time` 字段进行日期筛选，但 `ProcessRecord` 实体类中没有此字段。

**影响范围**:
- `/boss/dashboard`
- `/boss/kpi`
- `/boss/report/{id}`
- `/manager/dashboard`
- `/manager/report/list`

**代码位置**:
```java
// ProductionDashboardServiceImpl.java:214
wrapper.apply("DATE(record_time) = {0}", date);
```

**建议修复**:
- 方案1: 将 `record_time` 改为 `scan_time`（ProcessRecord实体中有此字段）
- 方案2: 在 ProcessRecord 实体中添加 `record_time` 字段

### 问题2: capacity_alert 表名不一致

**问题描述**: 
- `V2026.03.30__add_capacity_alert_tables.sql` 创建的表名是 `capacity_alert`
- `V2026.03.31__test_bug_fixes.sql` 创建的表名是 `jxc_capacity_alert`
- `CapacityAlert` 实体配置的表名是 `capacity_alert`

**影响范围**: `/boss/alert/list`

**建议修复**: 统一表名命名，建议全部使用 `jxc_` 前缀

### 问题3: year_month 保留字未正确处理

**问题描述**: MyBatis-Plus 在生成 INSERT SQL 时未正确转义 `year_month` 保留字。

**影响范围**: 工资单创建功能

**建议修复**: 
- 方案1: 自定义 mapper xml，手动编写 insert sql
- 方案2: 使用 MyBatis-Plus 的 `GlobalConfig` 配置自定义 SQL 生成器
