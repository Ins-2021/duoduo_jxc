## 四、发现的问题明细

### 问题1：数据库表结构不完整
- **严重程度**：高
- **问题描述**：初始化数据库时缺少多个业务表或表的列
- **涉及表**：
  - `jxc_purchase_order` 缺少 `discount_amount`, `other_fee`, `actual_amount`, `remark` 等列
  - `jxc_purchase_order_detail` 缺少 `create_by`, `update_by`, `deleted` 列
  - `jxc_inventory` 缺少 `available_qty`, `locked_qty`, `batch_no` 等列
  - `jxc_stock_in_out` 表不存在（已手动创建）
  - `jxc_inventory_check`, `jxc_transfer_order`, `jxc_inventory_alert` 等表不存在（已手动创建）
  - `jxc_inventory_transaction` 表不存在（已手动创建）
- **解决方案**：手动执行SQL脚本补充缺失的表和列

### 问题2：RBAC权限配置不完整
- **严重程度**：中
- **问题描述**：admin超级管理员角色缺少业务模块（采购、库存等）的权限
- **涉及权限**：`inventory:stock-in-out:add`, `inventory:transfer:approve`, `inventory:check:complete` 等
- **解决方案**：
  1. 在 `jxc_sys_menu` 表中补充缺失的权限记录
  2. 将权限分配给admin角色：`INSERT INTO jxc_sys_role_menu SELECT 1, menu_id FROM jxc_sys_menu WHERE perms LIKE 'inventory:%'`
  3. 清除Redis权限缓存：`DEL auth:perms:user:1`

### 问题3：出入库类型筛选问题
- **严重程度**：低
- **问题描述**：出入库查询接口传入type参数筛选时，返回了所有类型的记录
- **可能原因**：SQL查询中未正确使用type条件

### 问题4：进货退货接口缺失
- **严重程度**：中
- **问题描述**：系统中未发现独立的进货退货API接口
- **建议**：可通过新增负数数量的进货单实现退货功能

### 问题5：盘点完成权限不足
- **严重程度**：低
- **问题描述**：admin角色缺少 `inventory:check:complete` 权限
- **解决方案**：分配该权限给admin角色

## 六、备注

1. **数据库初始化问题**：系统启动时SQL初始化脚本未能完全执行，导致部分表缺失。建议检查 `application.yml` 中的 `spring.sql.init.schema-locations` 配置是否完整。

2. **权限缓存**：修改权限后需要清除Redis中的权限缓存才能立即生效。

3. **业务流测试限制**：由于库存初始为空，审核进货订单等依赖库存的业务流程无法完整测试。

4. **测试数据保留**：所有创建的测试数据（进货订单、入库单、出库单、盘点单、调拨单）均已保留在系统中，未删除。
