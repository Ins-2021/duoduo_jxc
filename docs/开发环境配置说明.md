# 开发环境配置说明

## 一、开发环境要求

### 1.1 后端环境

| 软件 | 版本 | 说明 |
|------|------|------|
| JDK | 17+ | 必须 |
| Maven | 3.8+ | 必须 |
| MySQL | 8.0+ | 必须 |
| Redis | 7.0+ | 必须 |
| IDE | IDEA 2023+ | 推荐 |

### 1.2 前端环境

| 软件 | 版本 | 说明 |
|------|------|------|
| Node.js | 18+ | 必须 |
| pnpm | 8+ | 推荐 |
| IDE | VSCode / WebStorm | 推荐 |

---

## 二、项目配置

### 2.1 后端配置

**application.yml**：

```yaml
server:
  port: 8080

spring:
  datasource:
    url: jdbc:mysql://localhost:3306/duoduo_jxc?useUnicode=true&characterEncoding=utf8
    username: root
    password: ${DB_PASSWORD:root}
    driver-class-name: com.mysql.cj.jdbc.Driver
    
  redis:
    host: localhost
    port: 6379
    password: ${REDIS_PASSWORD:}
    
mybatis-plus:
  mapper-locations: classpath*:/mapper/**/*.xml
  configuration:
    map-underscore-to-camel-case: true
    log-impl: org.apache.ibatis.logging.stdout.StdOutImpl
```

### 2.2 前端配置

**.env.development**：

```properties
VITE_API_BASE_URL=http://localhost:8080
VITE_APP_TITLE=多多进销存
```

## 三、数据库初始化

### 3.1 创建数据库

```sql
CREATE DATABASE duoduo_jxc 
DEFAULT CHARACTER SET utf8mb4 
COLLATE utf8mb4_unicode_ci;
```

### 3.2 执行初始化脚本

```bash
# 按顺序执行
mysql -u root -p duoduo_jxc < docs/sql/V1.0.0__init.sql
```

---

## 四、本地开发启动

### 4.1 后端启动

```bash
cd backend
mvn clean install
mvn spring-boot:run
```

### 4.2 前端启动

```bash
cd frontend
pnpm install
pnpm dev
```

### 4.3 访问地址

| 服务 | 地址 |
|------|------|
| 后端API | http://localhost:8080 |
| 前端页面 | http://localhost:5173 |
| API文档 | http://localhost:8080/doc.html |
| MySQL | localhost:3306 |
| Redis | localhost:6379 |

## 五、Docker环境（可选）

### 5.1 docker-compose.yml

```yaml
version: '3.8'
services:
  mysql:
    image: mysql:8.0
    environment:
      MYSQL_ROOT_PASSWORD: root
      MYSQL_DATABASE: duoduo_jxc
    ports:
      - "3306:3306"
    volumes:
      - mysql_data:/var/lib/mysql
      
  redis:
    image: redis:7.0
    ports:
      - "6379:6379"
      
volumes:
  mysql_data:
```

### 5.2 启动命令

```bash
docker-compose up -d
```

## 七、常见问题

### Q1: 端口被占用

```bash
# 查找占用进程
lsof -i :8080
# 终止进程
kill -9 {PID}
```

### Q2: 数据库连接失败

1. 检查MySQL是否启动
2. 检查用户名密码
3. 检查数据库是否存在

### Q3: Redis连接失败

```bash
# 启动Redis
redis-server
# 或使用Docker
docker run -d -p 6379:6379 redis:7.0
```
