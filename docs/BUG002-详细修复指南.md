# B002 Bug详细修复指南

## 问题描述
4个库存Service的update/delete方法未限制单据状态，已审核单据仍可被修改删除

## 涉及文件
1. `InventoryCheckServiceImpl.java`
2. `StockInOutServiceImpl.java`
3. `TransferOrderServiceImpl.java`
4. `AssemblyOrderServiceImpl.java`

## 修复原则

### 状态校验规则
- **update**: 仅允许草稿/待审核状态修改
- **delete**: 仅允许草稿/待审核状态删除
- **已审核/已完成状态**: 禁止任何修改

### 异常抛出
```java
throw new BusinessException(BizCode.BUSINESS_ERROR, "非草稿状态不允许修改");
```

## 修复示例

### InventoryCheckServiceImpl.java
```java
@Transactional(rollbackFor = Exception.class)
public void updateOrder(InventoryCheckDTO dto) {
    InventoryCheck exist = getById(dto.getId());
    if (exist == null) {
        throw new BusinessException(BizCode.DATA_NOT_FOUND);
    }
    
    // 状态校验 - 关键代码
    if (exist.getStatus() != InventoryCheckStatusEnum.DRAFT.getCode()) {
        throw new BusinessException(BizCode.BUSINESS_ERROR, "非草稿状态不允许修改");
    }
    
    BeanUtils.copyProperties(dto, exist);
    updateById(exist);
}

@Transactional(rollbackFor = Exception.class)
public void deleteOrder(Long id) {
    InventoryCheck exist = getById(id);
    if (exist == null) {
        throw new BusinessException(BizCode.DATA_NOT_FOUND);
    }
    
    // 状态校验 - 关键代码
    if (exist.getStatus() != InventoryCheckStatusEnum.DRAFT.getCode()) {
        throw new BusinessException(BizCode.BUSINESS_ERROR, "非草稿状态不允许删除");
    }
    
    removeById(id);
}
```

### StockInOutServiceImpl.java
```java
// 同样逻辑，使用StockInOutStatusEnum
if (exist.getStatus() != StockInOutStatusEnum.DRAFT.getCode()) {
    throw new BusinessException(BizCode.BUSINESS_ERROR, "非草稿状态不允许修改");
```
