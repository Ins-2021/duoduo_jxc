## 二、详细测试过程与结果

### 2.1 销售管理模块详细测试

#### 测试 1: jxc_receivable 表 create_by 字段

**测试步骤**:
1. 创建应收账单 (POST /finance/receivable)
2. 执行核销 (POST /finance/receivable/{id}/write-off)
3. 查询核销后的账单详情

**测试结果**:
```json
// 核销前 (receivableId=1)
{"amount":5000.00,"receivedAmount":0,"remainingAmount":5000.00,"status":0}

// 执行核销 1000 元后
{"amount":5000.00,"receivedAmount":1000.00,"remainingAmount":4000.00,"status":1}
```

**结论**: ✅ 核销功能正常，但 `create_by` 字段未自动填充（为 null）。实体类继承自 `BaseEntity` 包含 `createBy` 字段，但数据库列可能未正确创建或填充逻辑未实现。

---

#### 测试 2: 销售订单金额自动计算

**测试步骤**:
1. 创建销售订单，包含商品明细:
```json
{
  "orderType": 1,
  "customerId": 1,
  "operatorId": 1,
  "details": [
    {"skuId": 1, "spuId": 1, "warehouseId": 1, "qty": 10, "unitPrice": 100.00},
    {"skuId": 2, "spuId": 2, "warehouseId": 1, "qty": 5, "unitPrice": 200.00}
  ]
}
```

2. 查询创建后的订单

**测试结果**:
```json
// 订单详情 (orderId=1)
{
  "orderId": 1,
  "docNo": "XS202603311774922166930",
  "orderType": 1,
  "customerId": 1,
  "status": 10,
  "totalAmount": 0.00,       // ❌ 应为 2000
  "discountAmount": 0.00,
  "actualAmount": 0.00,      // ❌ 应为 2000
  "paidAmount": 0.00,
  "remark": "测试销售订单"
}
```

**预期**:
- 明细1: lineAmount = 10 × 100 = 1000
- 明细2: lineAmount = 5 × 200 = 1000
- totalAmount = 1000 + 1000 = 2000
- actualAmount = 2000 - 0 = 2000

**结论**: ❌ 未修复。`SalesOrderServiceImpl.createOrder()` 中虽有金额计算逻辑（第201-216行），但计算结果未正确保存。可能原因:
1. `converter.toEntity(dto)` 先复制了 DTO 到 Entity（此时 details 为 null）
2. 金额计算在 save 之前，但后续有其他逻辑覆盖了金额
3. 数据库表 `jxc_sales_order` 缺少 `total_amount`, `actual_amount` 等列

---

#### 测试 3: 销售订单审核流程

**测试步骤**:
1. 调用订单详情接口 (GET /sales/{id})

**错误信息**:

```
     delivery_qty, unit_price, discount_amount, line_amount, warehouse_id, ...
FROM jxc_sales_order_detail WHERE deleted = 0 AND (order_id = ?)
```

**结论**: ❌ 阻塞。`jxc_sales_order_detail` 表缺少 `delivery_qty` 列。

#### 测试 5: 核销记录生成

**测试步骤**:
1. 应收核销: POST /finance/receivable/1/write-off?amount=1000
2. 应付核销: POST /finance/payable/1/write-off?amount=500
3. 查询核销记录: POST /finance/write-off/page

**测试结果**:

应收核销前后:
```
核销前: paidAmount=0, remainingAmount=5000, status=0
核销后: paidAmount=1000, remainingAmount=4000, status=1 ✅ 正确
```

应付核销前后:
```
核销前: paidAmount=1000, remainingAmount=2000, status=1
核销后: paidAmount=1500, remainingAmount=1500, status=1 ✅ 正确
```

核销记录查询:
```json
{"total":0,"list":[]} ❌ 无记录
```

**代码分析**:
`ReceivableServiceImpl.writeOff()` 第136-145行有插入逻辑:
```java
com.duoduo.jxc.entity.WriteOff writeOff = new com.duoduo.jxc.entity.WriteOff();
writeOff.setWriteOffNo("HX" + ...);
writeOff.setType("RECEIVABLE");
writeOff.setBillNo(entity.getBillNo());
writeOff.setBillId(entity.getReceivableId());
writeOff.setAmount(amount);
writeOff.setCreateTime(LocalDateTime.now());
writeOffMapper.insert(writeOff);  // 插入但无记录
```

**结论**: ❌ 未修复。核销逻辑执行成功（金额更新正确），但 `jxc_write_off` 表无数据。可能原因:
1. `jxc_write_off` 表在数据库中不存在
2. `WriteOffMapper.insert()` 静默失败
3. 事务回滚导致插入被撤销
