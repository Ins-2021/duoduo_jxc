## 处理策略

根据评估文档和实际情况，采用以下处理策略：

### 高优先级（实际可用）
- ✅ **生产看板** - 已修复，接入真实数据
- ✅ **分类导入** - 移除导入按钮（数据量小，手工维护更可靠）

### 中优先级（需要开发）
- **工资单打印** - 使用系统现有打印服务
- **生产订单导入** - 从销售单导入
- **打印模板预览** - 使用现有打印服务

### 低优先级（可暂缓）
- 裁床管理相关（cut-bundle, cut-order）
- 生产排程、生产计划
- 返工管理、质检管理
- 预约单管理

## 具体处理措施

### 1. 移除低价值按钮
对于数据量小、使用频率低的功能（如分类导入），直接移除按钮

### 2. 禁用未开发功能
对于暂时无法开发的功能：
- 添加 v-if="false" 隐藏
- 或添加提示"功能即将上线"

### 3. 实现关键功能
对于老板最关注的功能：
- 工资单打印
- 生产订单从销售单导入
- 打印模板预览

### 4. 占位页面组件化
创建通用的 PlaceholderPage 组件，支持：
- 功能名称显示
- 预计上线时间
- 返回按钮
