### 2.1 销售订单金额自动计算 ❌ 未修复

**测试步骤**:

```json

    {"skuId": 1, "spuId": 1, "warehouseId": 1, "qty": 10, "unitPrice": 100.00},
```

2. 查询创建后的订单

**测试结果**:
```json
// 订单详情 (orderId=3)
{
  "orderId": 3,
  "docNo": "XS202603311774923693872",

  "status": 10,

  "discountAmount": 0.00,
```

**结论**: ❌ 未修复。虽然 `SalesOrderServiceImpl.createOrder()` 第201-216行有金额计算逻辑，但计算结果未正确保存到数据库。

### 2.5 账户余额更新 ❌ 未修复

**测试步骤**:
1. 查看账户初始余额: Account ID=1 → balance=10000.00
2. 创建收入单: amount=1000.00
3. 审批收入单: `POST /finance/income-expense/6/approve`
4. 再次查看账户余额

**测试结果**:
```
审批前余额: 10000.00
审批收入:   1000.00 (type=1, categoryId=1, accountId=1)
审批后余额: 10000.00  ❌ 应为 11000.00
```

**代码分析**:
`IncomeExpenseServiceImpl.approve()` 第118-133行有更新账户余额逻辑:
```java
if (entity.getAccountId() != null && entity.getAmount() != null) {
    FinanceAccount account = financeAccountMapper.selectById(entity.getAccountId());
    if (account != null) {
        BigDecimal newBalance = account.getBalance() != null ? account.getBalance() : BigDecimal.ZERO;
        if (entity.getType() == 0) {
            newBalance = newBalance.add(entity.getAmount());  // 收入加
        } else {
            newBalance = newBalance.subtract(entity.getAmount()); // 支出减
        }
        account.setBalance(newBalance);
        financeAccountMapper.updateById(account);
    }
}
```

**结论**: ❌ 未修复。代码逻辑正确，但余额未更新。可能原因:
1. `entity.getType()` 判断问题（收入 type=1，但代码判断 type==0）
2. 账户查询失败但未抛异常
3. 更新语句未实际执行

## 六、代码问题分析

### 6.1 销售订单金额计算问题

**文件**: `SalesOrderServiceImpl.java` 第201-216行

```java
// 自动计算订单金额
if (dto.getDetails() != null && !dto.getDetails().isEmpty()) {
    BigDecimal totalAmount = BigDecimal.ZERO;
    for (SalesOrderDetailDTO d : dto.getDetails()) {
        if (d.getQty() != null && d.getUnitPrice() != null) {
            BigDecimal lineAmount = BigDecimal.valueOf(d.getQty()).multiply(d.getUnitPrice());
            d.setLineAmount(lineAmount);  // DTO设置正确
            totalAmount = totalAmount.add(lineAmount);
        }
    }
    order.setTotalAmount(totalAmount);  // Entity设置正确
    ...
}
order.setActualAmount(totalAmount.subtract(order.getDiscountAmount()));
save(order);  // 保存后查询为0
```

**可能原因**: `converter.toEntity(dto)` 在第191行先复制了DTO到Entity（此时details为null），金额计算在第201行进行，但可能存在Converter覆盖或其他问题。

### 6.2 账户余额更新问题

**文件**: `IncomeExpenseServiceImpl.java` 第119-133行

